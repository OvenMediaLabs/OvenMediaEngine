# Building OvenMediaEngine on macOS (Apple Silicon)

Supported platform: **macOS 13+, Apple Silicon (arm64)**

---

## 1. System Prerequisites

### Xcode Command Line Tools

Required for the compiler (`clang`) and SDK headers. If you have Xcode installed this is already present; otherwise install the standalone CLI tools:

```bash
xcode-select --install
```

Verify:
```bash
xcode-select -p   # should print /Library/Developer/CommandLineTools or similar
clang --version   # should print Apple clang 15+
```

### Homebrew

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

After install, follow the printed instructions to add Homebrew to your PATH (typically `eval "$(/opt/homebrew/bin/brew shellenv)"`).

### Homebrew packages

```bash
brew install cmake ninja pkg-config nasm automake libtool xz
```

| Package | Purpose |
|---------|---------|
| `cmake` | Build system (3.24+ required) |
| `ninja` | Fast build backend |
| `pkg-config` | Library discovery |
| `nasm` | Assembler for FFmpeg/OpenSSL |
| `automake` / `libtool` | Required to build several C libraries from source |
| `xz` | Archive decompression during dependency build |

---

## 2. Install External Dependencies

All third-party libraries (FFmpeg, SRT, OpenSSL, whisper.cpp, etc.) are built from source and installed to a user-writable prefix.

```bash
# Default prefix: ~/opt/ovenmediaengine  (no sudo needed)
/opt/homebrew/bin/cmake -P cmake/InstallPrerequisites.cmake
```

To use a custom prefix:

```bash
/opt/homebrew/bin/cmake -DOME_DEP_PREFIX="$HOME/myprefix" -P cmake/InstallPrerequisites.cmake
```

This step only needs to be run **once** (or again when upgrading a library version in `cmake/Versions.cmake`).

> **Note:** The script detects macOS automatically and skips Linux-only components (NVIDIA stubs, `sudo`, etc.).

---

## 3. Configure

```bash
# Debug build (default)
cmake -B build/Debug -G Ninja

# With a custom dependency prefix
cmake -B build/Debug -G Ninja -DOME_DEP_PREFIX="$HOME/myprefix"

# Debug + unit tests
cmake -B build/Debug -G Ninja -DOME_BUILD_TESTS=ON
```

> **Do not** pass `-DOME_HWACCEL_NVIDIA=ON` on macOS — there is no CUDA on Apple Silicon. Hardware acceleration on macOS is via VideoToolbox (not yet wired up in the build).

---

## 4. Build

```bash
cmake --build build/Debug --parallel $(sysctl -n hw.ncpu)
```

The binary is placed at `build/Debug/bin/OvenMediaEngine`.

---

## 5. Run

```bash
# Copy example configs (first time only)
mkdir -p build/Debug/bin/conf
cp misc/conf_examples/Server.xml build/Debug/bin/conf/
cp misc/conf_examples/Logger.xml build/Debug/bin/conf/

# Run
build/Debug/bin/OvenMediaEngine
```

## 6. VS Code Integration

### Recommended extensions

- **CodeLLDB** (`vadimcn.vscode-lldb`) — required for `"type": "lldb"` launch configurations
- **clangd** (`llvm-vs-code-extensions.vscode-clangd`) — uses the `compile_commands.json` generated by CMake

Install via terminal:
```bash
code --install-extension vadimcn.vscode-lldb
code --install-extension llvm-vs-code-extensions.vscode-clangd
```

### Generating compile_commands.json (for clangd)

```bash
cmake -B build/Debug -G Ninja -DCMAKE_EXPORT_COMPILE_COMMANDS=ON
```

A `compile_commands.json` will appear in `build/Debug/`.

## 7. Known Differences from Linux

| Area | Linux | macOS |
|------|-------|-------|
| Linker | GNU ld (`--start-group`) | Apple ld64 (`-all_load`) |
| epoll | `epoll_*` syscalls | kqueue wrapper in `epoll_wrapper.h` |
| `sem_init` | unnamed semaphores supported | not supported → `condition_variable` |
| `pthread_setname_np` | `(pthread_t, name)` | `(name)` — current thread only |
| `high_resolution_clock` | alias for `system_clock` | alias for `steady_clock` |
| `TCP_QUICKACK` / `SOL_TCP` | available | not available — guarded |
| `SIOCINQ` / `SIOCOUTQ` | available | not available — guarded |
| `malloc_trim` | glibc only | not available — guarded |
| Hardware acceleration | NVIDIA / QSV / XMA | not yet supported |
| jemalloc | supported | supported (included in dependencies) |

## 8. Troubleshooting

**`CMake Error: generator Ninja does not match Unix Makefiles`**

A previous build used a different generator. Clear the cache:
```bash
rm -rf build/Debug/CMakeCache.txt build/Debug/CMakeFiles
```

**`Cannot create directory /opt/...: Permission denied`**

Use a user-writable prefix:
```bash
cmake -B build/Debug -G Ninja -DOME_DEP_PREFIX="$HOME/opt/ovenmediaengine"
```

**`Configured debug type 'lldb' is not supported` in VS Code**

Install the CodeLLDB extension:
```bash
code --install-extension vadimcn.vscode-lldb
```

**Test binaries time out during build (`gtest_discover_tests`)**

macOS scans newly compiled binaries on first execution (Gatekeeper), which can exceed a short timeout. The build system already sets a 30-second discovery timeout on macOS. If it still times out, run the binary once manually:
```bash
./build/Debug/bin/ome_test_base --gtest_list_tests
```
Then rebuild — the binary will already be cleared and run fast.
